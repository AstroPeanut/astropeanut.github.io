Kelley Hess在组会上的报告

发表于 2017-03-16 | 分类于 工作和技能 | 0 comments

Kelley Hess在今天的组会上做了个关于CHILES数据处理方面的报告，此页是我总结的笔记。

数据处理部分

介绍了一下CHILES观测的一些基本情况。摘录如下：

  • 1002h观测时间，VLA的B configuration
  • 946-1426 MHz，对应的HI为红移0到红移0.12
  • channel宽度是62.5kHz，这对应于红移0.12为15km/s
  • 预计能探测到约300个HI rich objects

观测的最近的update

  • phase 1 (13B) 178h (13TB的最终科学数据产出)
  • phase 2 (15A) 213h (数据处理中)
  • phase 3 (16B) 211h (数据处理中)

    预计还需要两轮观测来完成survey,后续的观测正在申请

数据处理update

  • 预先的60个小时观测数据已经处理并发表了，见(Fernandez et al. 2013)
  • phase 1的完整480MHz带宽的数据都已经处理并成图完毕
  • 在整个带宽上noise比较平坦，观测质量良好
  • 部分频带受RFI影响很严重，我们依然在努力搞好这块
  • imaging, continuum subtraction依然很挑战

SOFIA和source finding

SoFiA有两种工作模式

  • blind search
    • 事实证明现在的SOFIA还无法自己很好的完成工作。如果选择了blind模式的话，结果并不好看。有很多false positive或者true negative。不能用这个。
  • 输入预先已知的源
    • 可以先经由其他波段的已知catalog，作为输入。这个是可行的模式。

用SOFIA做的source finding测试得出了如下结论：

  • Optical_find.py
    • cube越小，false positive情况出现的越少
    • cube越小，用来决定是否是false positive用到的统计越差
    • 改变搜寻的cube大小会改变‘探测’的结果
    • 这个merging parameter要反复敲打才能找到合适的，这个参数设小了会把展源拆成多个，设大了也不行会把false positive也连接进来
  • 使用建议
    • 计算noise的时候，与其使用原先的那个特定的area，应该给一个更大的区域。（而且这个区域最好是没有什么源的）
    • 不要把positive和negative源merge到一起

几种情况。

  1. SOFIA能干好，人眼也能干好的情况
    这个不消说，是一些比较明显的源，相对比较亮的。
  2. SOFIA干的好，人眼干不好的情况
    比如，有个旋涡星系，其边缘很延展，但是很暗弱，人眼难以分辨出来边缘的feature，但是SOFIA可能结合多个channel的一些统计量进行评估，进而把边缘的一些信息解析出来。
  3. SOFIA干不好，人眼能干好的情况
    典型的比如sidelope比较明显，但是在不同channel处这些sidelope表现不一样，本来看单个channel数据是看不出来的，而结合了多个channel的统计信息SoFiA可能会更加懵逼。这个情况人眼就容易一些，毕竟sidelope不管怎么飘，切换channel总能发现一堆sidelope围着一个不动的信号源。

科学

由于已经处理完的数据只有phase 1的，所以所谓科学产出目前只能从这里来。

  • 在pilot的60个小时观测那篇文章中，发布了红移0~0.18范围内的33个candidate detections.
  • 红移0.12探测到了15个星系
  • pilot中的6个红移在0.12的candidate已经得到了证实
  • 探测到了红移0.17处两个在void里的星系

观测到的groups

  • 44个质量大于$10^{13}M_\odot$的Knobel groups
