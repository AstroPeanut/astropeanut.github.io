u盘安装双系统到dell机子上的boot error解决

发表于 2017-03-14 | 分类于 工作和技能 | 0 comments

如题，这个机子可能比较老了，是师兄临走的时候留下来送给我的。我一开始只打算往里面装个ubuntu，结果各种不行。网络检索后尝试安装不同的桌面环境，unity不行就上gnome，都不行。最后测试了xfce桌面的ubuntu（即xubuntu）才成功的。然后因为最近用deepin用的比较顺手而且体验确实也不错，于是想往那个机子里面把linux系统换成deepin。结果尝试了很多很多次，花了很多时间，无果。前后断断续续也折腾了好一阵子，最终明白过来了原因，把结论记录在此留个存照。

深度官网提供了u盘启动制作程序，我为了省事儿也就一直用那个。然后每次尝试引导机子都直接提示boot error读不出来。我今天打算往那个机子里面装个win7，为了省事儿，直接用那个深度的启动程序制作启动盘了，一引导，发现出现了boot error。我一下子明白过来咋回事儿了，问题不出在机子，也不出在u盘，而是出在深度这个启动盘制作程序本身——因为我的win7已经装了无数次肯定没毛病。于是我直接在mac上开win7虚拟机，用软碟通软件制作引导盘。结果都一次成功了。
所以我终于肯定下来问题所在。这个机子可能比较老，支持的u盘启动文件格式比较有限。我进入boot menu发现u盘项里面只有USB-ZIP.这就意味着USB-HDD, USB-HDD+, USB-ZIP+很可能都引导不了。而深度这个启动软件为了简便，根本没有设置这个的步骤，全部预设好了。八成它是给设置成USB-HDD啥的了。因而导致启动失败。用软碟通的话，制作前设置为USB-ZIP，就解决了。

ps，其实u盘装系统真的很快。俩系统，一共花了也就40分钟不到。
