通勤路线坡度调研小记

发表于 2017-02-17 | 分类于 生活和提高 | 0 comments

刚搬了家，新家好处确实很多，但毕竟是远了。谷歌地图查了下大概是4公里多。
这就比较蛋疼了，房租是便宜了一些，但是距离确实远了不少，步行肯定是不行了。在Stirling Hwy上，公交极其方便，但是无比蛋疼的是Perth的公共交通是按”zone”收费的，家刚好在zone1的边上，要收跨区公交费，贵的一比，更不要想每天都要坐两趟了。
所以争取每天骑自行车是相对现实的路子。这个距离骑车还是比较合适的，但是骑车要考虑的事儿就比较多了。除了天气因素，我最直接的担心是上坡下坡的问题，于是有了这篇文章。

谷歌地图上给的结论是“大多是平坦的道路”，但是我之前有一次骑车往西边走过大概一公里多，累成狗，所以对谷歌的说法简直是“信了你的邪”。Google搜索Maps Altitude，第一个点进去，发现可以实现我查询海拔的目的。
操作很简单，先输入家庭位置，找到。沿着路在地图上点击，下面的数据框就会出现相应的经纬坐标和海拔等信息。它可以同时输出距离信息，但是错的太离谱了，得自己算。。
我一共点了大概100个点，在数据框中把经纬度、海拔复制并格式化粘贴到excel里面，然后再粘贴到一个文本文档里面。
然后要做的事儿就是把经纬度转换成为实际距离并作为横轴，纵轴是海拔。
经纬度转换成实际距离的原理实际上不难，最直观的简单算法就是先假定地球是正球体（这个假设不过分）然后用余弦定理来算。但是因为我这个4km距离相对于地球的半径太小了，算cos的时候误差会比较大所以不是个好办法。其实想想也知道这个肯定在各种语言里面都有现成的包来计算了，但是还是比较好奇具体算法所以进行了一些调研。有一篇比较有趣的是美团外卖写的技术文档，我觉得以后有兴趣可以进行一点相关研究，尤其是那个Haversine公式，还没有弄明白咋回事儿。

备查，家的坐标是-31.98254,115.77723000000003，海拔20.4米。ICRAR的坐标是-31.977704714583545,115.81546783447266，海拔11.7米。以及，用IDL直接用map_2points命令输出这两点的距离，显示只有3.65km的直线距离，我还是挺吃惊的。。

用IDL写很简单，不过一个map_2points()命令而已。我的电脑idl的license有问题，于是在服务器上跑的。

跑出来两张图：

程序中计算的dis_total确实是3.6公里，而最后的dis_sum显示为4公里多一点。这个和谷歌地图给出的是一致的，我之前认为这个路径比较直了，看来还是不够笔直所以多出来了几百米。
