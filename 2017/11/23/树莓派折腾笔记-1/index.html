树莓派折腾笔记-1

发表于 2017-11-23 | 分类于 兴趣爱好 | 0 comments

洗份儿给买了个树莓派做生日礼物，开心。最近一两天花了很多时间在这个上面，架设了几个基础服务，也踩了一些奇奇怪怪的坑，不过总体上来说还是比较友好的，毕竟，linux嘛..

目前没有做硬件的想法，毕竟对模电数电不够熟悉，后面的引脚短期内不打算使用。想要个树莓派还是看中了其功耗低可以$7\times 24$工作的好处。

目前其负担的服务有

  • 配合音响使用的网易云音乐服务
  • 一个微信聊天机器人服务
  • 一个twitter机器人服务
  • 一个家庭文件服务器NAS
  • 一个kodi家庭影院
  • 一个远程下载机服务

本文的目的是简要记录一下这几个服务的架设过程。期间踩过一些坑，本着拿来主义的原则搜索了大量的网页信息，所以无法完全恢复问题解决的过程甚至是方法，仅根据记忆做一下记录。

配合音响使用的网易云音乐服务

正好家里面有一对不用的小音响在那里吃灰。毕竟是个音响嘛，音质还是可以的。但是毕竟平时都直接用电脑听歌了，开音响多此一举于是它就在那儿吃灰了…看了下我这个树莓派3b是有3.5mm耳机接口的，那就简单了，直接接上。

但是如何听歌这个却折腾了一小会。开始先把音响放在客厅电视柜下面，dvd机的两边，而树莓派一小个电路板而已我就给放在dvd机的后面，还挺合适。先开豆瓣fm听了一上午，感觉很不错，洗份儿也表示很惊喜。但是为了听歌开个谷歌浏览器实在是太浪费了，监控cpu占用一直挺高，于是想想怎么办。

github上面有个比较老的项目，叫做musicbox，是命令行模式的网易云播放器。这个牛人最近开发了一个用微信作为遥控器的程序，用网易云的api和itchat这个api搞的，这个挺吸引我的。但是似乎这个项目还不是很成熟，反正我搞了一会没搞定，果断放弃，转念用musicbox，就按照github给的文档安装，很顺利。使用体验很棒，毕竟一般我在客厅放音乐的场景，也是我躺在沙发上工作的时候，抱着电脑，iterm留一个小窗口给musicbox方便的很。

唯一的一个问题，是因为版权问题，歌单遇到灰色的歌曲就直接卡在那了，过个一分钟左右才开始下一个。有时连续好几首歌都是灰的，它就要卡好一会。这个有点烦，目前不知道怎么解决。

一个微信聊天机器人服务

直接找的教程，用itchat，全是api，一共也写不了几行程序。。这里把小坨叽的号贡献出来了。接入的是图灵机器人服务，免费版。

python脚本放在了桌面上，启动树莓派的时候运行一下就行了。暂时没有设置开机启动服务，怕遇坑。

可以实现简单的聊天功能，比如问天气，查快递啥的。当然日常聊天就别想了，亲测跟siri差不多的水平，付费的应该会好些？

在图灵机器人的portal界面，发现可以自己提供知识库。尝试教他一些个别的词条，比如对“你是谁呀”的回答设置成“我是小坨叽呀，我有个粑粑麻麻U•ェ•*U”这种。然后跟它微信聊天，发现确实更改了。但是“你是谁”的回答依然是之前的那个，添加“你是谁”的回答也无效。我怀疑当知识库产生冲突的时候，我添加的就被忽略掉了…于是我尝试查看默认的知识库，看能不能定制一下，把一些我觉得不好的回答改一下，没搞定，不知道怎么修改。

另一个有意思的是，知识库是可以批量导入的，免费服务账号一天也有上千条知识库导入的配额。无奈暂时不知道这些知识库去哪里找的到。无论如何，现在的这个机器人日常聊天的时候还是笨笨的那种，跟siri差不多德行。。

一个twitter机器人服务

twitter服务机器人就更简单了，当然保险起见我还是先建了个小号。申请api接口的话要求必须有手机绑定。
同样的，写了个非常简单的python脚本，可以实现发一些简单的twitter，比如hello world这种，也可以推一些预设文件夹下的图片。总体来说还是个只会hello world的机器人，相当于只是测试通过了api的使用。剩下的就是点子+实现了。

一个点子是每天早晨发一条twitter报告一下天气，晚些时间可以发送个“历史上的今天”的推，晚上可以随机发送一条笑话这种。这个自然要用到cron计划任务了。计划任务应该不难，但是实现点子目测需要其他的api以及一些数据库的知识，所以以后再说。

一个家庭文件服务器NAS

这里用到的是samba。

实现的结果是，用mac在局域网内用finder在shared一栏可以直接看到树莓派挂载的3TB的硬盘。点击直连(当然要输入个预设的用户名和密码)，然后可以直接访问里面的所有文件，爽歪歪。值得一提的是，亲测硬盘里面的视频也可以直接双击打开播放。

一个kodi家庭影院

原来叫xbmc，现在改成kodi了。算是一个开源的电视盒子软件，国内的小米盒子等智能电视用户也又不少刷它的。
安装没啥说的，我用的debian系统，直接上apt-get就行了，但是配置比较麻烦。

这里有两件事儿比较重要：

  1. 实现手机做遥控器
  2. 下载和使用插件，以便正常访问国内的电视直播等资源

关于第一点，在app store里面搜索kodi就能找到kodi remote.
第二点，毕竟人在国外，youtube啥的也犯不着用它来看，如果能看youku搜狐斗鱼哔哩哔哩等的话，才真的美滋滋。

现在的情况是，去下载了大量的插件，但是没一个能用的。比如优酷tv，在选好了电视剧，点击播放之后，会提示网址解析失败。排查发现这些错误应该还是因为ip段不在国内，版权原因遭到屏蔽了。那么我的解决方案应该是找个全局代理，或者至少是kodi局部代理的方式。国内有两个vps，一个是linux的一个是win7的，说不定可以派上用场。anyway，以后再说吧，毕竟也没那么大的欲望非得看国内的电视剧。。

然后呢，我的那个kodi remote总是闪退，所以也没法使用。。。一个猜测是我的手机太老了…软件比较新兼容性差？毕竟ip5么..

值得一提的是，kodi做为单纯的视频播放器还是很出色的。它作为电视盒子固件的初衷，体现在当运行kodi的时候默认界面会投放在电视机上面，但是桌面依然在。啥意思呢，teamviewer连接树莓派，开终端敲kodi之后，作为显示器的彩电会直接跳入kodi界面，但是teamviewer里面依然是桌面环境，干别的事儿不耽误。我scp进去了几个b站下载的视频，播放了一下，体验还是很不错的。

一个远程下载机服务

先是尝试使用迅雷的远程下载服务，毕竟看到大部分人都是这么干的。
也很顺利。主要是找那个固件花了一点时间。。。解压到硬盘，运行那个potal，生成激活码，去迅雷远程的网址，登录，绑定即可。
我发现没有速度，以为必须得离线才有速度，就去买了个15块的会员，结果mmp没效果。反正我到现在都没能让它正常下载，要么就显示下载机不在线，要么就是没反应。搜索了半天没看到类似的问题，所以还是没解决。。

尝试了一下Aria2，配置到一半放弃了。
然后没办法了找了个土办法，反正我有teamviewer，相当于有显示器键盘鼠标了，直接操作图形界面的机子也是可以的。于是去下载了个deluge，直接图形界面操作。但是尝试了几个种子，速度上不来，普遍是两三百k这样。

然后试了下transmission。作为下载机，使用的是transmission-daemon，教程很多。配置完后会有个网页端口，局域网所有机器可以访问，查看核控制下载。目前的测试结果是，非常好用。速度很快，操作方便，优雅。

一些坑以及一些想法

我现在树莓派默认开机挂载一个64G的优盘，作为其硬盘使用。sd卡系统盘只有8G，目前剩余1.7G希望不会很快用完。。另外就是看需求手动挂载3T的硬盘。这个硬盘是要插电的那种，那么功耗就必须要考虑了，毕竟我买树莓派就是要省电的。。调研了一下，这个硬盘的功耗还是挺可观的，估计在6-7W这个样子，比树莓派本身的功率绝对是要高的。看评测，树莓派的工作功率一般不会这么高。

折腾的过程中，出现了几次机器无法启动的情况：都是出现在修改fstab文件以使得开机自动挂载外部磁盘。往往是没有给合理是设置所致。如果这个文件里面给添加的磁盘设置不当，就会导致无法启动系统。然后要么就格式化重新装系统（第一次这个故障我就是重装了系统），要么就把tf卡拿出来用电脑读取，修改command.txt，在后面加上init /bin/sh。然后放回树莓派并且通电。这样在启动的时候会给一个root的shell环境。在这个时候尝试去修改fstab文件为正常的即可。

不过我遇到的情况是无法修改fstab，提示为已读。

网络给出的解答是，尝试mount一下/根目录。我试了一下发现失败，最后有一个stack overflow上没有被顶的很高的家伙，说你得mount到一个其他的地方，比如/sda6啊这种。因为要不然默认会加载原来的fstab配置，依然会出错。因为emergency mode下挂载的盘是只读的无法修改。
