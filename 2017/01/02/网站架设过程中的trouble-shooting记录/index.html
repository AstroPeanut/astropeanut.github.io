网站架设过程中的trouble shooting记录

发表于 2017-01-02 | 分类于 工作和技能 | 0 comments

一开始是因为在国内有了个免费的VPS，通过不断网络搜索大概明白了可以用它来做的事情。翻墙是一个，不过往回翻意义总是小一点。另外一个就是建站。看了些介绍发现建站其实很容易，如果比较幸运没出什么岔子的话一两个小时搞定，不过我显然属于不幸运的那一类。TT 因为比较折腾，所以把这一过程记录下来很有必要，防止以后踩坑。

在狗爹godaddy上买了一年的域名，很便宜，8块人民币。然后在github上研究博客。期间捉摸了hexo等工具，比较折腾。现在还是在github上玩着，熟悉了以后看看能不能在VPS上搞。（不知道会不会有政策方面的因素要考虑？估计有备案什么的事儿。）
另一方面，写博客也确实有其显而易见的好处。我最近一两年使用OneNote和Evernote记录了很多东西，但是很零碎。我觉得其中一个中啊哟原因，是由于私人化写作容易过于随意而难成体系，博客是公开化的，下笔要考虑到有人看到，某种程度上对自己是种规范。要求提高了，下笔就要稍微考虑一下了。
我初步打算在这个博客上记录的内容有，天文知识，技术文档，艺术评论等。

安装hexo

博客得好看而且容易操作，所以用了hexo，我的一半时间花在着了折腾这个上面，所以我主要记录一下这里踩的坑。

另外在mac平台下node.js可以直接下载在图形界面安装。我在这里折腾了特别久。原因就是npm安装hexo这一步老是出错。网上有不少人遇到和我一样的问题，但是我的用他们的方法解决不了。后来有些博客说这个node.js最好在nvm下安装。最终是从官方文档解决的问题：https://hexo.io/docs/。其中关键一个是nvm install stable.通过nvm ls可以查看目前使用的版本，之前存在的奇奇怪怪的问题，在按文档插入运行nvm install stable之后都正常了。另外要注意的是，每次新开一个终端窗口，hexo命令都会失效，所以每次开终端以后都要运行一下nvm install stable，然后可以正常使用hexo相关命令。
